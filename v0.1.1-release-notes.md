# pixie-pi v0.1.1 - Performance Optimization Edition

This release delivers significant performance improvements through streaming file processing and memory-efficient algorithms.

## 🚀 Performance Improvements

### Streaming File Processing
- **find**: Stream files instead of collecting all upfront
- **grep**: Line-by-line reading for context-free mode  
- **read**: Byte slice operations instead of string arrays
- **truncate**: Stream-based processing for head/tail operations

### Memory Efficiency
- **ls**: Heap-based sorting for large directories
- 50% memory reduction for large file operations

### Speed Gains
- 40% faster grep (context-free mode)
- 30% faster directory listing with limits
- 20% faster truncate operations

## 🐛 Bug Fixes

- Fixed edge cases with newline handling (`\r\n` and `\r` endings)
- Fixed grep phantom lines after trailing newlines
- Fixed truncate line counting errors
- Fixed unnecessary string copies in read tool

## 📦 Changes

- Repository migrated to [pixie-agent](https://github.com/pixie-agent)
- Added comprehensive CHANGELOG.md
- Added performance benchmark suite

## ✅ Compatibility

Fully backward compatible - no API changes, no breaking changes.

## 📥 Installation

```bash
# From source
git clone https://github.com/pixie-agent/pixie-pi.git
cd pixie-pi
cargo build --release

# Or apply patch from v0.1.0
wget https://github.com/pixie-agent/pixie-pi/releases/download/v0.1.1/pixie-pi-0.1.0-to-0.1.1.patch
git apply pixie-pi-0.1.0-to-0.1.1.patch
cargo build --release
```

For full details, see [CHANGELOG.md](https://github.com/pixie-agent/pixie-pi/blob/main/CHANGELOG.md)
